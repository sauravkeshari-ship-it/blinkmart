BlinkMart

Quick-commerce apps made delivery fast — but building the cart is still painfully manual. BlinkMart AI changes that with a conversational assistant that understands intent, generates a full cart instantly, and lets users refine via natural language. From search-first to intent-first shopping.

Splash Screen
SCREEN 01

Splash Screen

The yellow brand screen establishes BlinkMart's energetic identity. Bold typography — "Blink" in black, "Mart" in green — creates instant brand recognition while communicating freshness and commerce.

Brand IdentityTrust Building
  • Brand yellow (#F8CB46) conveys warmth and optimism
  • Plus Jakarta Sans ExtraBold for maximum impact
  • Version number builds transparency and trust
  • Auto-transitions to Home via zoom — minimal friction
SCREEN 02

Home — The Familiar Base

Conventional grocery app patterns — categories, flash sales, promos — so users feel instantly comfortable. After 1 second on the home screen, a popup card rises above the AI button with a transparent overlay — introducing the assistant without leaving the shopping context.

Assist, Not ReplaceAI Discovery
  • Manual shopping remains fully functional — AI is optional
  • Blink AI popup auto-appears after 1s — gentle AI introduction
  • Centered AI button with pulse animation draws attention
  • Transparent overlay keeps home visible — low disruption
  • Three suggestion chips bridge the "what can I ask?" gap
SCREEN 03

Education — Onboarding the AI

When users first tap the AI button, a bottom sheet introduces Blink AI with three concrete examples. This bridges the "what can I even ask?" gap — the biggest barrier to AI adoption in commerce apps.

Soft PushReduce Anxiety
  • Educating user about New feature an what they can do with it
  • Three real examples show AI capabilities concretely
  • Budget-aware query example sets expectations right
  • "Tap to Explore" CTA is low-commitment
SCREEN 04

Chat Mode — AI Greeting

The dark interface signals "AI mode" — a clear visual break from the commerce UI. Five suggestion chips eliminate the blank canvas problem, giving users ready-made starting points from real user research.

User ControlDual Input ModeAlways Editable
  • Dark theme = AI mode, Light theme = commerce mode
  • 5 chips based on actual user research personas
  • Text input + voice input available simultaneously
  • Back arrow ensures escape hatch to manual shopping
SCREEN 05

Chat Flow — Intent to Cart

The user types "Fruits under ₹200" and the AI responds with a structured product card. "Best Match" label shows confidence, the total stays under budget (₹195 < ₹200), and every item is individually adjustable.

Confidence TransparencyReview Before Add
  • User intent → structured result in one exchange
  • "Best Match" confidence label builds trust
  • Budget respected: ₹195 total under ₹200 limit
  • Individual quantity controls on every item
  • "Add to cart" requires explicit user action
SCREEN 05

Immersive Voice Mode

Full-screen dark interface with an ambient wave visualization. Designed for hands-free, on-the-go shopping. The gradient wave provides visual feedback that the system is listening.

Voice-First UXImmersive Experience
  • Full-screen removes distractions for focused interaction
  • Wave animation = visual feedback that AI is listening
  • Dark gradient signals premium AI experience
  • Two clear actions: mic (continue) and close (exit)
SCREEN 05.1

Voice — Speak, See, Confirm

Same conversation in voice mode — text appears as transcript alongside the product card. Results use the same card format, ensuring consistency across both input modes.

Show Your WorkMode Consistency
  • Voice input transcribed in real-time for verification
  • Same product card component used in both modes
  • Wave animation continues during processing
  • User can switch to manual editing at any point

SCREEN 06

Result — The Confidence System

User clicks on view all button in AI list card to get here, Three tabs — Best Match, Suggested, Alternative — are the trust architecture's heart. Users see why each item was picked and explore alternatives. No black box.

Confidence Transparency3-Tier System
  • Best Match: Highest relevance + budget fit
  • Suggested: Good alternatives for variety
  • Alternative: Swap options and budget adjustments
  • Search icon allows manual search fallback
SCREEN 07

Checkout — Familiar Territory

Intentionally conventional. After AI cart creation, users land on a standard checkout they trust. Wallet, delivery, products, payment — no surprises.

Assist, Not ReplaceTrust Continuity
  • Standard checkout pattern — zero learning curve
  • Wallet balance shown upfront for transparency
  • Delivery address editable — user stays in control
  • Total matches AI's calculated value — builds trust
SCREEN 08

Payment Selection

Standard payment selection with multiple options. "Frequently Used" puts wallet front and center — reducing taps. Green highlight on selected method provides clear feedback.

Reduce FrictionStandard Patterns
  • Most-used payment method auto-highlighted
  • Green border signals active selection clearly
  • Multiple methods: wallet, PayLater, cards, bank

Payment Success

Green checkmark with confetti celebrates completion. The entire flow — "Fruits under ₹200" to payment — took under 2 minutes. 60% faster than manual cart building.

Emotional DesignGoal Achievement
  • Green checkmark = universal success indicator
  • Confetti particles add celebratory micro-delight
  • Clear next action: "Back to Home"
